MightyTwin · Mobile UX Redesign

A coherent, top-to-bottom mobile experience built on three reusable patterns — persistent shell tabs, a per-pane bottom carousel, and an expandable mini-widget mode for in-place tools. Borrows useful conventions from apps like AllTrails (carousel-of-cards, pull-up sheets) but keeps Twin's existing desktop chrome — sidebar, right pane, dock — intact. Each mobile pattern below has a corresponding desktop surface listed underneath.

Pattern key

Shell tab bar
Persistent Map · Atlas · Settings at the very bottom (already shipped by app-shell). Same on every screen.
Top brand bar
48px. Just the "MightyTwin" wordmark + dev-only breakpoint chips. Identity only — no critical controls (so iOS banner notifications dropping in at the top don't eat anything functional).
Primary controller pill
Horizontal floating pill anchored top-left of the map. Two states: Overview (no site loaded) → "All sites · N" label with a stack-of-maps avatar. Site loaded → just the site chip (no back affordance here — that lives at the start of the widget rail instead). Right of the chip: zoom + home + bounds + basemap controls.
Overview tile (back-to-overview)
Square 48×48 icon-only button at the start of the widget rail when a site is loaded. Violet tint signals it's a navigation action (not a tool). Tap → back to all-sites overview. A subtle 1px separator divides it from the widget tiles. Doesn't appear in Overview state (you're already there).
Site list strip
In Overview state the bottom of the map shows a horizontal site picker (cards scroll-snap, peek-of-next, click to load). Replaces the widget rail one-for-one — widget rail is per-site so it doesn't appear until a site is loaded. Same data + same component on every form factor; just sized differently.
Bottom carousel
Horizontal scroll, snap-x, hidden scrollbar. On Map it's the widget controller (Story/Snap/Design/Terrain/Fly). On Atlas/Settings it's the section nav. On Sites it's a content carousel of site cards.
Widget sheet (decoupled)
Tapping a carousel tile launches a sheet that slides up over the carousel and the shell tab bar — the widget gets the full lower viewport rather than fighting the carousel for vertical space. Height is dynamic per widget-state: a single quick action collapses to ~280px; an active Design session can grow past 600px. Map + ctrl-pill stay visible above for spatial context. Pull-down or tap-close returns to launcher (carousel + tabs visible again).
Drill-in
Tapping a content card (e.g. a site in Atlas → Sites) pushes a full-pane detail view. Brand bar gets a back chip. Bottom carousel can persist or hide depending on the destination.
Floating left-edge icons
Slim vertical stack on the Map canvas — only the actions NOT already in the primary controller pill. Currently: Search, Measure, Legend. No layers/basemap/3D buttons — those live in the pill. Replaces ViewerSidebar's column from desktop.
Mai FAB
Bottom-right, persistent. Default Y clears the 64px bottom rail (88 px above the shell tab bar). Hides during forced-breakpoint preview and while a tools sheet is open.

Open questions to confirm before code

Launch gesture
Tapping a tile launches the widget sheet directly — no drag-up handle on the carousel itself. The carousel becomes purely a launcher; the sheet is the workspace. Sheet's own handle handles resize / dismiss.
Sheet height policy
Sheet sizes to its content per widget-state, capped at min(content, ~92vh) with a sensible minimum (~240px so single-action widgets don't look cramped). User can drag the handle to override within those bounds. Pull-down past a threshold dismisses entirely.
Widget sheet contents
Each widget renders its own sheet body — same component, but with mobile-first density. Proposed registry slot: widget.MobileSheet on the widget extension. Existing desktop right-pane render stays untouched.
Atlas Sites carousel
Wholly new for mobile, optional for desktop. Sources its data from the existing /api/sites endpoint. Tap a card → drills into Map mode with that site loaded.
Back-to-overview
Small chevron left of the site avatar inside the primary controller pill (frames 1, 2). Tap → navigates to /viewer (all-sites overview). The whole site chip can double as a site-switcher sheet.
Right-pane on desktop
Unchanged. The widget sheet is phone-only — desktop continues to dock widget content in the right pane. Same widget components, two render targets (widget.MobileSheet vs widget.RightPane).
