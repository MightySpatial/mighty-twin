Where everything lives

  • Brand zone (top-bar left): complementary hierarchy. When a client logo is set (Settings → Workspace admin → Branding), the client wordmark is primary (15 px name + 26 px coloured mark) and Mighty drops to a small "by Mighty" attribution behind a hairline divider. Unbranded workspaces show MightyTwin as primary. The two states share classes; a .branded modifier flips weight.
  • Site chip: embedded into the top bar (not floating over the map). Hero chooser opens a popover with current/recent sites + "+ New".
  • Layout slider + Settings: top-bar centre + right (already shipped).
  • Vertical zoom column: left edge — zoom in/out · home · 2D/3D · basemap. Active state = vertical bar accent (UX rule #6).
  • Nav gimbal: top-right corner of the map surface. Compass with N/E/S/W ticks, heading arrow, tilt arc. Click resets to north + level. Replaces Cesium's default gimbal which we'd been hiding.
  • Legend: off by default; per-site Atlas setting. The Legend tile in the primary rail toggles it on/off for the current session. When enabled for a site (Atlas Site Layout Designer → Legend visible), it docks at a configurable corner (top-right under gimbal by default). Twin doesn't expose a legend on every map automatically because most engineering sites don't need one; clean by default, on-demand when it matters.
  • Time slider: appears as a strip ABOVE the bottom rails when temporal layers are on. Hides when no temporal data is active. Inset-left so it clears the scale bar.
  • Transparency: NOT a top-level widget. Lives inside the Layers panel (per-layer slider), since transparency is meaningful per layer not globally. Global "see-through-the-globe" is its own thing in the Layers panel header.
  • Primary widget rail (cobalt-tinted): Search · Measure · Layers · Legend. Always visible, the four tools every user reaches for. The Legend tile is a session toggle; per-site default-on/default-off is set in Atlas.
  • Secondary widget rail (sits above primary): Story · Snap · Design · Table · Strike · Terrain · "More ▾". Same tile shape, neutral background. The "More ▾" expands a popover with the long tail.
  • Per-widget config: each widget registers with { id, label, controller: 'primary' | 'secondary' | 'none', position: number, loadMode: 'floating' | 'sharePane' | 'drawer' | 'inline', defaultSize, scope: 'global' | 'per-site' }. Settings → Engine → Widgets layout edits the global catalog; Atlas Site Layout Designer overrides per-site. loadMode matters:
    • floating — free-floating panel anchored on the map (Layers panel default).
    • sharePane — share the right rail with AI chat: widget stacks above chat, chat shrinks to bottom half. Keeps the map surface uncluttered when several widgets are open at once.
    • drawer — slide-up bottom drawer (Attribute table default).
    • inline — fixed map-edge component (zoom column, gimbal, scale bar — not user-toggleable).
  • Feature popup: ANCHORED to the click point with a leader line + teal pulsing dot. Smart placement avoids the gimbal / chat / rails. Phone variant slides up from bottom as a 60 % sheet.
  • AI chat: minimised by default until configured (slim 56 px right column with vertical "Mighty AI" label, pulsing violet sparkle, "setup" badge). Collapses/expands via the chevron in the header. State persists across sessions — once configured + expanded, it stays expanded.
  • Phone: bottom tabs (Map / Atlas / Settings), two FABs at the bottom corners — Tools (left, slate) and AI (right, violet). Site chip becomes the header. Gimbal + zoom column live top-left/top-right inside the map surface.

Settings → per-widget config (forward-looking)

Each widget registers with: { id, label, primary | secondary, defaultPosition, defaultSize, scope: 'global' | 'per-site' }. Settings → Engine → Widgets layout shows the catalog and lets admins toggle primary/secondary tier and default size. Per-site overrides land later via the Atlas Site Layout Designer.
