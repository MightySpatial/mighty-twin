In-viewer tool behaviors

How each voxel tool looks on the globe. Each card is a stylised Cesium-canvas snapshot showing the cursor state, ghost preview, gizmo handles, and selection chrome the user sees while the tool is active.

Block paint cursor + grid-snapped ghost
BLOCK PAINT · 1.0 m · ROCK

Cursor + ghost preview

The cursor snaps to the hover-block's lattice. A 50%-opacity teal ghost outlines where the next click will paint. The material colour fills the ghost so the user reads "what will appear" before clicking.

Generator placement 3D gizmo handles
PYRAMID · base 14 × 14 · h 7 · α 45°

Per-handle direct manipulation

An arrow gizmo controls height; corner handles drag the base rectangle; an arc handle on the wall edge sets the inset angle. The PlaceModeBar inputs and the gizmo are bidirectional — dragging a handle updates the input value and vice versa.

Terrain mask live block fill under polygon
TERRAIN MASK · drawing polygon · 4 verts

Polygon draw with live block preview

The user clicks vertices on the globe. As each vertex lands, the engine tessellates the polygon, samples the terrain surface at the chosen block level, and renders the block fill underneath in real time. The double-click closes the polygon and commits.

Water fill click point + elevation slider
WATER FILL · 42.5 m AHD

Floating elevation slider near the cursor

After the user clicks a datum point, a vertical slider docks near the cursor. Dragging it raises or lowers the water plane; the fill animates to track. A flat surface line extends out to the basin's natural extent.

Block selection hover · selected · marquee
SELECT · 14 blocks · ROCK + ORE

Three selection states

Indigo outline on hover, teal outline + soft fill on selected, dashed teal marquee while drag-selecting. Multi-select uses ⇧ + click; the marquee adds whatever it overlaps.
