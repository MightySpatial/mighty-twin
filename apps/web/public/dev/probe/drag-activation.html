Pointer state machine

Probe drag

  1. pointerdown on Probe rail tile → glyph spawns at pointer position; tile renders as "ghost" (dashed border).
  2. pointermove on document → glyph follows pointer; on every move, scene.pick(windowPosition) hit-tests the map.
    • If hit returns a primitive tagged navigable → glyph turns solid indigo, cursor = grabbing.
    • Else → glyph stays semi-transparent gray, cursor = not-allowed.
  3. pointermove also broadcasts a probe:drag-hover event so the map can highlight the candidate space (glow + outline).
  4. pointerup:
    • Over a navigable feature → fire probe:activate with the feature's NavigableSpace id + drop point. Camera flies to the nearest centerline point.
    • Over empty / non-navigable area → glyph animates back to the tile (200 ms ease-out bounce). Toast: "Drop on a navigable feature."
    • Over a non-navigable feature, user is admin → inline prompt offers the Mark-navigable modal with the feature pre-selected.
  5. Escape key during drag → cancel drag, glyph returns to tile.

Tap fallback (no drag, just tap the tile)

  1. If pointerdown → pointerup with movement < 6px in < 250ms → treated as a tap.
  2. Tap → entire map enters "tap to probe" mode (same highlight + cursor as drag).
  3. Next tap on a navigable feature → activate. Tap elsewhere → exit mode silently.
  4. Escape key → exit mode.

Street View drag — same pattern

  1. Same pointer state machine. Different glyph (Google's pegman SVG, blue when over coverage).
  2. Drop validity = StreetViewService.getPanorama(point, radius) resolves with a panoId.
  3. Coverage hint layer = Google's StreetViewCoverageLayer, drawn under the cursor only during drag.

Accessibility

  1. Tile has role="button" aria-label="Probe — drag to a feature on the map".
  2. Tap fallback ensures users without precise pointer control (or screen-reader users) can still activate.
  3. Keyboard activation: Tab to tile, Enter → "tap to probe" mode, ↑↓←→ moves a focus reticle over the map, Enter selects.
